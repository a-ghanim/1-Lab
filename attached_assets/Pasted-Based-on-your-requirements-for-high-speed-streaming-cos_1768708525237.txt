### Why this fits your specific features:

* **p5.js Simulations:** Gemini 3 Flash is highly capable of generating clean, functional JavaScript. For complex physics simulations, you can enable the **"medium" thinking level** via the API to ensure the logic is sound before the code is streamed.
* **Structured JSON Output:** Both models natively support `response_mime_type: "application/json"`. You can provide a JSON schema in your API call to ensure the quizzes and curricula always match your platform's data structure without parsing errors.
* **Document Summarization & Annotation:** With a **1-million-token context window**, you can upload entire textbooks or long research papers. Flash can "read" these in seconds to generate annotations or summaries.
* **Streaming & Cost:** Gemini 3 Flash is optimized for the "Live" experience. It supports fast, server-sent events (SSE) for streaming, and its pricing is a fraction of the Pro model, making it the only viable choice for a "high volume" user base.

### Strategic Implementation Tip

For the best balance of quality and cost, use a **Router Pattern**:

1. **Flash (`gemini-3-flash`)**: Use for 90% of tasks—quizzes, summaries, progress tracking, and simple curricula.
2. **Pro (`gemini-3-pro`)**: Use only for the initial "Deep Dive" curriculum generation or when a user asks for a particularly complex simulation (e.g., "Simulate a double-pendulum with drag").
